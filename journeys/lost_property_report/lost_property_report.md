Lost Property Report — Non‑Offense Workflow

Purpose

This document outlines requirements and recommendations for Lost Property Report cases in OTIS 2.0. Lost property incidents cover situations where property belonging to the Ohio State Patrol (OSP) or the public is missing without a criminal element (e.g. misplaced equipment, lost evidence).

Key Points

- Property Lost by OSP: Lost property cases often involve state‑issued equipment (badges, weapons, radios) or evidence mishandled by division personnel. OTIS should allow users to record these items clearly and indicate whether they are division property or personal property.
- Optional Person Association: A lost property case may not involve a specific person (e.g. a firearm discovered missing from an inventory). OTIS must allow the property tab to stand alone without requiring a person entry. However, it should support adding a person—such as the officer who lost the property—when appropriate.
- Multiple Items: When multiple items are lost in a single event (e.g. lost badge and firearm), users should be able to list each item separately but summarize them under one case. Consolidating multiple items reduces duplication and simplifies tracking.
- LEADS Reporting: Certain lost items require entry into LEADS (Law Enforcement Automated Data System) to flag them as missing. For example, a lost badge, firearm, or vehicle license plate must be recorded. OTIS should prompt the user to enter details required by LEADS and capture the LEADS entry reference number.
- Linking to Other Systems: If the lost item is eventually recovered, the case should be updated and linked to the recovery incident. Support cross‑referencing with recovered property cases to close the loop on property lifecycle.

Recommended Workflow

1. Incident Creation: Create a non‑offense case with nature code Lost Property Report (797). Add additional nature codes if other activities occurred during the same incident.
2. Property Entry: Use the property tab to record each missing item. Capture details such as serial numbers, make/model, description, and last known location. Indicate if the item is division property or privately owned.
3. Optional Person: If applicable, add a person (e.g. officer) and assign roles (e.g. Reportee or Involved Officer).
4. Narrative: Provide a succinct narrative describing the circumstances under which the property was discovered missing and any search efforts.
5. LEADS Entry: For items requiring LEADS reporting, record the LEADS entry number and date/time of entry. Attach any confirmation documentation.
6. Case Updates: If the property is subsequently found or recovered, update the case and link it to the recovery incident. Document resolution and disposition.

Notes

- Lost property cases should not require victims or suspects. They are purely administrative for tracking and auditing purposes.
- Training should clarify when to create a Lost Property Report versus Recovered Property or Stolen Property case.
